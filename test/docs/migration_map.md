# Legacy test and evaluation migration map

This is the sole canonical audit record in which historical sequence identifiers are retained. They are listed only to prove that every legacy asset was accounted for. Canonical filenames, manifest IDs, imports, and normal command output use semantic names.

Baseline commit: `921171ac1ef5c85ab5a7cd1882dd40e1471b79f0` (`HEAD == origin/main` before migration).

Baseline result abbreviations:

- `guard pass`: the real-tree static check exited 0 with no findings.
- `unit pass (N)`: the legacy unit script ran N tests and exited 0.
- `baseline defect`: the command ran and failed; the failure is preserved rather than reclassified.
- `CLI inventoried`: the tool had no dedicated legacy unit command and was inventoried/read instead.

The migrated results below are fail-closed outcomes from the post-review canonical
runner. The exact final command, commit, totals, and evidence directory are recorded
in `baseline_validation.md`; `FAIL` remains visible where the unchanged production
baseline violates an assertion.

## Assets intentionally retained outside `test/`

The following are build/development utilities, not test implementations, and remain under `scripts/`: `axconfig-gen.py`, `cargo-axplat.sh`, `ensure-cargo-vendor.sh`, `rust-objcopy.sh`, `render_project_document.py`, all `scripts/make/*`, and all `scripts/net/*`. Existing Rust crate unit tests and vendor upstream tests remain beside their crates because they are production-package tests rather than legacy OrayS test-infrastructure assets.

No GitHub Actions file is part of this migration. Linux differential testing is not introduced.
